## 3 · 怎么算(校准规则,已固化)

**能量侧**
- Active energy:HAE 原始 **×0.85**,取区间下端。
- BMR baseline:**~1900**(「只是活着」;保守 1750 与 Apple basal ~1991 之间)。
- net = 摄入 − TDEE;活动段 = TDEE − BMR(含 TEF 近似)。

**摄入侧(宁可估高,别低估)**
- 餐厅 +30%(看情况:炭烤/清淡少加)、家做 +20%、隐藏油糖 +10–20%。
- 习惯零食 buffer **+200/天**(彩虹糖/坚果/黑巧均值)
  — ⚠️ 不改变减脂速率(体重 trend 已含),只说明真实 TDEE 比 bottom-up 高 ~200。

**报告纪律**
- deficit 报区间**下端**(更小缺口);体重预测报区间**上端**(更高 kg);**永不单点**。
- 体重 = **trend** 不是单日(5/31 脱水 79.9 = outlier)。
- 实测锚:5/18–6/6 回算 **TDEE≈2712 / 摄入均 2081 / 缺口≈−631**。

## 3.5 · 每日热量结算 SOP(固定四步)

> 给某一天「结算」:摄入 − TDEE = 净值。

**① 摄入** — 读当天 Diet Tracker(Food Log 各餐 + buffer)。餐厅 +30%(炭烤/清淡少加)/ 家做 +20% /
隐藏油糖 +10–20%;习惯零食 **+200/天**。
**② 活动** — 有运动:读 `Workouts-YYYY-MM-DD`,`Active Energy (kJ) ÷ 4.184` = 运动净卡
(实测,⭐ 别凭感觉估 — 犯过 4+ 次)。无运动:活动段 ≈ NEAT 200–500。
**③ TDEE** = BMR **1900**(只是活着)+ 运动净卡 + 其它 NEAT(~200–250)+ TEF(~10% 摄入)。
**④ 净值** = 摄入 − TDEE;报区间**下端缺口**(更保守)。

*worked example(6/11 网球日):摄入 2355(果泥165+烤鸡950+蛋炒饭900+饮料140+buffer200)−
TDEE 2755(BMR1900+网球398实测+NEAT220+TEF237)= −400。*

⚠️ 净值是**当天**信号,体重才是结果;单日净值 × 多日体重 trend 一起看
(单看一个会骗:水分/糖原扰动单日体重,估算扰动单日净值)。

## 3.7 · ⭐ 脂肪 vs 水分解(体重去噪)

> 核心问题:scale 每天上下跳 ±0.5–1.5kg,但真脂肪每天只动 ~0.08kg。
> 怎么把「真脂肪趋势」从「水/糖原噪声」里分出。

### 物理锚(整套方法的地基)
- 1kg 脂肪 ≈ **7700 kcal**(Wishnofsky;现代动态模型视为长期近似、非每日精确常数)。
- 你缺口 ~600 kcal/天 → 每天最多掉 **~0.08kg 真脂肪**(=600/7700)。
- ⭐ 所以:**任何单日 >0.10–0.15kg 的体重变化,数学上必然是水/糖原/肠内容物,不是脂肪。**
  单日噪声 ~±0.5kg = 脂肪日信号的 ~6–10 倍 → 单日体重不携带脂肪信息,**只有 trend 携带**。

### 真实体重 = 去噪趋势
EWMA 指数加权(= TrendWeight / Hacker's Diet / MacroFactor 的 trend weight):
`Trend_今 = Trend_昨 + α × (Scale_今 − Trend_昨)`,α = 0.1–0.15。近端权重大、滞后小;
单个 refeed/flare 异常日要 flag 或用中位。
⚠️ EWMA 在持续下降序列上**系统滞后**(trend 坐数据上方、残差偏负)——causal 平滑固有性质,实时可接受。
**回溯分析改用 LOESS/居中**(无滞后、残差对称 = 真水噪声),artifact 的「真实体重」线就是 LOESS。

### 残差 = 水(分解的关键)
`残差 = Scale − Trend`。这串残差就是水/糖原/肠内容物噪声。
实测(LOESS):残差 ±0.3–0.6kg,SD 0.28,对称。

### 糖原-水(最大摆动源)
- 经典 **~3g 水 / 1g 糖原**(Olsson & Saltin 1970:碳水加载 4 天 +2.4kg 体重、+2.2L 总体水)。
- 总糖原 ~400–500g 肌 + ~100g 肝 → 带 **~1.5kg 水**;大幅耗/补摆动 ~2kg(= 赛前加载所见)。
- ⚠️ **3:1 情境依赖、有争议**:Fernández-Elías 2015 活检发现 1:3 只在脱水时成立;补足水时高达 1:17。
  **用 3:1 当下限,个人系数靠自己数据标定。**

### 钠 / 肌酸 / 训练 / 炎症(次级)
- **钠**:急性移水但噪声大(Heer 2000:高盐未必升体水—可 osmotically inactive 存皮肤)。
  → 「高/低」分类代理足够,精确 mg 是伪精度。
- **肌酸**:近恒定 baseline 偏移 ~0.5–1kg(Desai 2024 meta +0.86kg);已在吃 → 烤进 baseline,**不计每日**。
- **训练**:不习惯/离心 + 赛后糖原超补 → DOMS 潴水 +0.2–1.4kg(24–72h)。
- **HS flare**:炎症 + 皮质醇激活盐皮质激素受体 → 钠水潴留。**flare/硬训日当 outlier flag,别当平滑协变量。**

### A · 回溯标定(spreadsheet 配方)
1. **EWMA/LOESS 出 trend**。
2. **残差 = scale − trend**。
3. **回归**:`残差_t = β₀ + β₁·碳水_t + β₂·碳水_(t−1) + β₃·训练_t + β₄·高钠_t + β₅·flare_t + e`。
   含**前一天碳水**(碳水→水次晨才显);钠/flare 用 0/1。
4. **Newey-West/HAC 标准误**(残差自相关,普通 SE 会骗;R `sandwich::NeweyWest` / Python `cov_type='HAC'`)。
   验证:留出近 2–3 周看 adjusted 是否更准预测 trend;碳水系数对应的水:糖原比落 2–4:1 才合理。
5. **数据量**:10–20 obs/预测变量 → 6 变量需 **~60–120 天**。现 ~50 天 →
   先用 trend+物理锚(几乎不需数据),系数当临时,过 ~8–10 周再锁。
   缺重日:EWMA 顺延;回归直接丢该行。

### B · 每日校正(5 秒)
`校正体重 = Scale − [β₁(碳水_t−均) + β₂(碳水_(t−1)−均) + β₄·高钠 + β₅·flare + β₃·硬训]`
教科书默认(未个人标定前):
- 碳水:每 +100g over 均 → **−0.3kg**(+300g refeed ≈ −0.9kg);低碳/空腹久 +0.4。
- 高钠日:−0.3~0.7kg(最噪、争议)。
- flare/硬新训练:−0.3~1.0kg(或直接忽略该日读数)。
- 肌酸:不调(baseline 已含)。
⚠️ 单日校正仍 **±0.5kg**;价值在 adjusted 的 **trend**,别盯单日。

### 平台 vs 假平台(whoosh)
真平台和水掩盖的假平台,raw 上长一样。分解能分:
**adjusted trend 仍降 而 raw 停 = 还在减,停是水**(refeed/flare 后常见);
**adjusted 也平 10–14 天 = 缺口真没了**(代谢适应/热量蠕动)→ 行动。

### ⭐ 6/12 实算(教科书例)
- 真实趋势(LOESS)6/12 = **80.06**,scale 80.2,残差 +0.14 → 今天体重只有 80.1 是真的。
- 6/11 79.6(残差 −0.59,练球耗糖原=水低)→ 6/12 80.2(refeed 水回)。
  **+0.6kg 若是脂肪需 +4600kcal 盈余 → 不可能,是糖原-水。真实体重没弹。**
- 6 月初 raw ~80.8 看着平台,但那几天残差为正(水偏多)在掩盖;去噪 trend 一直 ~−0.55kg/周 降。
  **「漂到维持」是水假象,真减脂没停。**

## 3.8 · RED-S 能量可用性 EA(安全底,与 deficit 互补)

> deficit(摄入−TDEE)管减脂速度;**EA 管「运动后还剩多少能量给生理功能」**——
> 两者会分叉,高训练日可以总缺口不大但 EA 危险地低。RED-S/IOC 共识(Mountjoy 2014/2023)核心。

### 定义 + 阈值
- **EA =(摄入 − 运动消耗 EEE)/ 去脂体重 FFM**(kcal/kg FFM/天)。
- 阈值:**≥45 最优;30–45 reduced/亚临床;<30 低 EA(LEA,内分泌/骨/恢复受损);男性 <25 进风险区**
  (Loucks 女性 LH 阈 ~30;男性更抗、阈更低 ~20–25)。

### 你的数字(FFM 估 ~67kg,按 ~17% 体脂;⚠️ DXA 可精确,给准体脂重算)
- 休息日:摄入 ~2100 / EEE 0 → EA ≈ **31**(reduced 区,勉强)。
- 网球日:摄入 ~2355 / EEE ~400 → EA ≈ **29**(当天已 LEA)。
- 121/对抗日:EEE ~500–600 → EA ≈ **27**(逼近男性风险区)。

### 为什么重要(deficit 视角看不到)
- deficit 会说「网球日 −400 没事」;**EA 说「运动吃回太少,EA 掉 29」**。
- LEA → ① 削弱免疫/炎症调节 → **HS 更糟** ② 降 RMR(适应性产热)→ 平台 ③ 叠加 ADHD 漏吃。
  与「感染期缺口 cap −200~−300」同一保护逻辑。

### 规则(EA = 底线,deficit = 杠杆)
- **训练日把运动吃回,保 EA ≥30**(加 ~150–250 kcal,哪怕当天缺口变小);**赤字主要靠休息日攒**。
- 休息日可背更大缺口(无运动,EA 仍 OK);但守 ADHD 防漏底 1800–2000。
- 标定后:EA 进每日结算(SOP 第②步算 EEE 时顺手 /FFM 看 EA)。

## 5 · ⭐ Recharts 踩坑(必记)

**图元不能包在 Fragment 里。** recharts 扫描子组件找笛卡尔元素(Line / Bar / Area)时**不会展开 React.Fragment**
→ `{cond && <>…多个图元…</>}` 里的线 / 柱根本不渲染(而直接子级的体重线却正常显示——这就是 debug 线索)。
✅ 正解:每个条件图元都是**独立直接子级** `{cond && <Element/>}`,绝不分组进 Fragment。

## 6 · 设计原则(通用)
- 富数据但要**清楚**(过密 = 不直观,过简 = 太简单);层级 banner → tile → detail → action。

- 移动端单列 max-width ~470–480;**禁 localStorage**。
- 引用放对话(cite 标签),artifact 内放纯文本。
