# UPCSE · 统一生理-认知状态估计

> 把所有 HAE 数据源(睡眠 / HRV / HR / 能量 / 训练)+ 饮食 + 主观语气,融合成
> **5 个可解释分数 + 5 个生理潜变量** 的个人状态引擎。
> 定位:WHOOP / Oura 级别,但**更科学、更透明、能跑在 n=1 稀疏数据上**。

UPCSE = **U**nified **P**hysiological–**C**ognitive **S**tate **E**stimation。
来源:逆向 WHOOP / Oura / Garmin-Firstbeat / Polar + 运动生理 / 神经科学文献。取数解码见 `file-operations.md`。

## 0. 目的与定位
- **痛点**:子页各管一摊(睡眠、减脂、晨报),但「我现在整体状态如何、今天能不能高强度用脑 / 打球」没有单一答案。
  商业设备(WHOOP / Oura)给一个黑箱分数,但 ① 不透明 ② 不为我个人校准 ③ 不整合饮食 / 主观。
- **UPCSE = 上层整合器**:把所有子系统输出汇成一个状态读数。
  两条铁律压倒一切 —— **输出准确度第一** + **完全透明可解释**(每个分数都能追到证据 + 权重)。
- **横跨**:睡眠和恢复 / 减脂引擎 / HRV / 训练负荷 / 炎症 / 认知 / 主观语气 —— 全部喂进来。

## 1. 设计哲学(逆向 WHOOP / Oura / Garmin / Polar 得出)
- **两层结构**:不直接出分数。先估 **5 个生理潜变量**(latent,不可直接测),再由潜变量合成 **5 个分数**(score,给人看)。
  分数变化永远能解释成某个潜变量动了。
- **个人基线 + 偏离即信息**:HRV / RHR 高度个人化 → 一切对**自身 ~28 天基线**归一(z 分),不用人群常模。
  偏离基线本身携带信息(↑↓ 都有意义)。
- **主观 > 客观(敏感度)**:Saw / Main / Gastin 2016 系统综述 —— 主观自评对训练负荷变化的**敏感度高于**客观指标
  → 语气 / 自评**重权**。
- **贝叶斯精度加权融合**:每个潜变量由多个有噪声的源估计 → 按**逆方差(精度)加权**合并;源越可信权越大。
- **优雅降级**:缺数据时**拓宽方差、不臆造插值**(missing ≠ 0)→ 分数置信度自动下降,但系统不崩。

## 2. 架构:5 潜变量 → 5 分数

### 5 个生理潜变量(latent · 0–100)

| 潜变量 | 含义 | 主要来源 |
|---|---|---|
| **自主神经平衡** Autonomic | 副交感 vs 交感张力 | HRV(SDNN)、RHR、日间 HR |
| **能量可用性** Energy / 糖原 | 糖原储备 + RED-S 能量可用性 | 摄入、运动消耗、体重 trend |
| **睡债** Sleep Debt | 累积睡眠压力(两过程模型) | 睡眠时长 / 结构史 |
| **炎症 / 生病** Inflammation | 亚临床炎症 / 感染征兆 | HRV 骤降、RHR 升、呼吸、HS、主观 |
| **认知储备** Cognitive Reserve | PFC 可用执行资源 | 睡眠 + 能量 + 压力 + 主观专注 |

### 5 个分数(score · 0–100,给人看)

| 分数 | 回答 | 由哪些潜变量合成 |
|---|---|---|
| **恢复** Recovery | 身体恢复到几成 | 自主神经 + 睡债 + 炎症 |
| **压力** Stress(高 = 差) | 当下生理负荷 | 自主神经(反)+ 炎症 + 能量赤字 |
| **睡眠** Sleep | 昨夜睡得如何 | 睡债 + 睡眠结构 |
| **就绪** Readiness | 今天能扛多少体能负荷 | 恢复 + 能量 + 训练负荷(ACWR) |
| **认知** Cognitive | 今天能扛多少高认知用脑 | 认知储备 + 睡眠 + 能量 + 压力(反) |

## 3. 各模块方法 + 研究地基

### 3.1 HRV 归一(→ 自主神经)
- 正道:**ln(RMSSD)**,7 天滚动均 + **±0.5×SD 最小有意义变化(SWC,Plews & Buchheit)** 判方向。
- ⚠️ **Apple 导出的是 SDNN 不是 RMSSD**(SDNN 含更多呼吸 / 动作成分)→ 已知妥协;
  日间 SDNN 噪声大,**只看趋势不看单点**。
- 个人基线 **65–85ms**;剔 >2× 中位伪迹。

### 3.2 训练负荷(→ 就绪)
- **Banister TRIMP**(时长 × HR 储备加权强度)量化单次内部负荷。
- **EWMA ACWR**(急性:慢性负荷比,指数加权):甜区 **0.8–1.3**,>1.5 伤病 / 过载风险。

### 3.3 糖原 + 能量可用性(→ 能量)
- 糖原:运动按碳水氧化耗损;补充 **7–10 g/kg/天 → ~24h 填满**(肌糖原 ~400–500g)。
- **RED-S 能量可用性 EA** =(摄入 − 运动消耗)/ 去脂体重(kcal/kg FFM/天):
  **≥45 最优,<30 低 EA(LEA),男性 <25 内分泌 / 情绪风险**。减脂期 EA ~30(可接受但需守底)。详见 `weight-energy.md` 3.8。

### 3.4 睡眠 PSG 校正 + 两过程(→ 睡债)
- Apple 低估深睡(**Stucky 2024:深睡灵敏度 50.5%、−43min**)→ 深睡 **+43min 校正 + 拓宽不确定度**。
- **两过程模型**:Process S(同态睡眠压力)+ Process C(昼夜节律)→ 估累积睡债。

### 3.5 HRV → 炎症 / 生病(→ 炎症)
- **Hasty 2021:HRV 较基线骤降 >40% → 对 CRP 升高约 91% PPV**(亚临床炎症 / 感染前兆)。
  RHR 同步升、呼吸升为佐证。HS flare 也喂此项。

### 3.6 主观 / 语气(→ 认知 + 压力)
- **Saw / Main / Gastin 2016**:主观自评敏感度 > 客观 → 重权。
- **LIWC**(语言计量)从对话语气提情绪 / 激活度;**30 秒自评(疲劳 / 心情 / 压力 各 1–5)是最高精度锚**。

### 3.7 贝叶斯精度加权融合
- 每潜变量 = 多源加权:`latent = Σ(精度ᵢ × 估计ᵢ) / Σ精度ᵢ`;精度 = 1 / 方差。
- 缺数据 → 该源方差 → ∞、权 → 0,其余源接管;**不插值**。

## 4. 计算流程(透明,bash 实算)

1. **取数**:HAE JSON / CSV(睡眠 / HRV / HR)+ 饮食实记 + 语气。解码法见 `file-operations.md`。
2. **估 5 潜变量**:各模块 → z 分 → 0–100。
3. **合成 5 分数**:潜变量加权(权重显式写死,可审计)。
4. **出 dashboard**:日间为主、夜间为辅;缺数据标灰。

```python
# UPCSE 计算核心(权重示意,与 6/12 结果一致;实际另有不确定度传播)
# 1) 潜变量(各模块 → z分 → 0-100)
A = autonomic(hrv_sdnn, rhr, hr_day)        # 56  自主神经
E = energy(intake, expend, weight, EA)      # 70  能量/糖原
D = 100 - sleep_debt(sleep_history)         # 78  睡债(高=债少)
I = inflammation(hrv_drop, rhr, resp, hs)   # 10  炎症(低=好)
R = cognitive_reserve(D, E, stress, subj)   # 80  认知储备

# 2) 分数 = 潜变量加权(权重显式)
Recovery  = 0.45*A + 0.30*D + 0.25*(100-I)               # 69
Stress    = 0.45*(100-A) + 0.30*I + 0.25*deficit         # 28 (高=差)
Sleep     = 0.60*D + 0.40*structure                      # 90
Readiness = 0.40*Recovery + 0.35*E + 0.25*acwr_fit       # 74
Cognitive = 0.40*R + 0.25*D + 0.20*E + 0.15*(100-Stress) # 76

# 3) 贝叶斯融合: latent = Σ(精度ᵢ·估计ᵢ)/Σ精度ᵢ; 缺数据→方差→∞→权→0
```

## 7. 局限(诚实)
- **没有设备直接测皮质醇 / 糖原 / 认知** —— 这三个全是**建模代理推断**,不是测量。
- **WHOOP / Oura / Garmin 内部专有算法是逆向重构**(近似,非原版)。
- **Apple 导 SDNN 非 RMSSD**;日间 SDNN 动作噪声大、单点不可信。
- **n=1,非因果**;分数是**状态估计不是诊断**。

## 8. SOP & 经验
- 出分数前**先取真实数据**(HAE JSON / CSV + 饮食),别凭感觉估潜变量。
- 新数据多是**确认**而非大改 → 别为了「动」而编精度变化(改 1 分是噪声)。
- **日间生理**:watch 白天戴着 HRV 一直在记;CSV 里 HR 可能有空档(8:31→14:40 离腕),用 HRV 补。
- 最高精度提升 = **30 秒主观自评**,直接锁认知 + 压力(主观敏感度 > 客观)。

## 9. 实时多通道状态(2026-06-16 实装)· 计算方法全集

> 来源:逆向 WHOOP / Garmin-Firstbeat / Oura / Polar + DOMS / 神经肌肉 / 情感神经科学文献。
> **核心修正**:§2 的「5 分数合成」会把恢复时间线**不同**的系统压成一个就绪分,从而**掩盖背离**
> (典型:HRV 绿灯、但腿 DOMS 9/10)。→ 部分维度**保持独立通道、不合成**,与 5 分数**并行**。

### 9.0 不合成原则(为什么)
各系统恢复时间线不同 —— 自主神经 ~24h、神经肌肉 ~48h、DOMS 峰 24–48h / 消退 5–7d、CK 4–5d。
**合成单一分会掩盖可操作背离**。→ **5 个独立通道**,各带「保真度」标签(高 / 模型 / 空)。

### 9.1 实时心肺 %HRR(保真度:高)
- `%HRR = (HR − RHR_floor) / (HRmax − RHR_floor) × 100`
- 参数:**HRmax = 190**、**RHR_floor = 45**(≈月均静息,待动态化)、储备 RES = 145
- **非代谢 HR** = HR 高于动作预测且持续 >5min → 认知/情绪唤醒代理(见 §9.5)

### 9.2 训练负荷 CTL / ATL / TSB(保真度:高·客观实测)
- **负荷单位 = Apple Watch 实测 Active Energy(kcal/日)**,客观完整;多 session 当天相加;无 workout = 0。
- 有 HR 的天用 **Banister TRIMP** 交叉验证:`TRIMP = D·x·0.64·e^(1.92·x)`,`x = (HRavg−HRrest)/(HRmax−HRrest)`
- ⚠️ **手表自 ~6/8 起停记运动 HR** → 近期回退 active energy;修复后可恢复 TRIMP。
- **EWMA**:`CTL_t = CTL_{t-1} + (load−CTL_{t-1})/42`;`ATL_t = ATL_{t-1} + (load−ATL_{t-1})/7`;
  `TSB = CTL − ATL`;`ACWR = ATL/CTL`(甜区 0.8–1.3,>1.5 警戒)
- ⭐ 修正示例:6/4 4h(1789 kcal)客观负荷 > 6/13 SS#3(1591)—— 时长占主导,RPE 估计低估长 session。

### 9.3 肌肉酸痛 DOMS(保真度:模型)
- `S(t) = A·(e^(−t/τd) − e^(−t/τr))`,归一到峰值 = A;**τr = 16h、τd = 55h**;单峰 t* ≈ 28h;约 6–7 天消退
- A 由 eccentric 负荷 × novelty 驱动,**自动从日历反推、无需打分**;分部位 腿/肩/核心;多事件叠加 clamp 10
- ⚠️ **酸痛 ≠ 力量恢复**(解离)→ 必须独立通道

### 9.4 隔夜恢复锚(保真度:高)
- `z = (x − 基线均)/基线SD`,逐指标 HRV(SDNN)/RHR/呼吸/睡眠;方向:HRV、睡眠 高=好,RHR、呼吸 高=差
- **≥2 项朝坏方向偏离 >1 SD 才触发**;`SWC = 0.5 × CV`(Plews & Buchheit);HRV 7 天滚动、每周 ≥3 采样
- 本周静态基线:**HRV 72±8 / RHR 45±3 / 呼吸 14.2±0.6 / 睡眠 7.0±0.8**

### 9.5 心理 · 唤醒(保真度:部分)
- **唤醒可读**(非代谢 HR + HRV);⚠️ **效价 valence 不可读**:压力 vs 兴奋同一自主画像;
  需面部 EMG(corrugator/zygomaticus,Sato 2021)或 EEG → 手表无 → 须主观标记 → **不追踪**。
- **神经肌肉(空)**:HRV 测不了(r 与力量 p=0.05–0.98)→ 需 CMJ 纵跳 → 用户选免 → **不追踪**。

### 9.6 与 §2「5 分数」关系(并行互补)
- 5 分数答「整体几成」;5 通道答「**哪个系统拖后腿**」(尤其 DOMS/神经肌肉被就绪分吸收会丢失)。
  数据源共享,差别在聚合方式。
